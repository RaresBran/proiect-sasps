# Studiu comparativ între arhitectura monolitică și cea bazată pe microservicii în dezvoltarea unei aplicații web

Proiectul propus are ca scop realizarea unui studiu comparativ între arhitectura monolitică și cea bazată pe microservicii, prin dezvoltarea unei aplicații web simple de tip Task Manager (gestionare a listelor de activități). Scopul principal este evidențierea diferențelor în ceea ce privește organizarea codului, performanța, mentenabilitatea și scalabilitatea între cele două abordări arhitecturale. Aplicația va permite utilizatorilor să se autentifice, să creeze, să editeze și să șteargă task-uri, să le clasifice pe categorii și să vizualizeze statusul acestora.

Pentru implementare, se va utiliza un stack tehnologic ușor și portabil, bazat pe limbajul Python și framework-ul Flask, ales pentru simplitatea configurării și flexibilitatea în crearea de API-uri REST. În varianta monolitică, toate componentele — interfața, logica de business și accesul la date — vor fi integrate într-o singură aplicație. În varianta bazată pe microservicii, aplicația va fi împărțită în servicii independente: un serviciu pentru autentificare (auth-service), unul pentru gestionarea task-urilor (tasks-service), și un serviciu simplu pentru interfață sau agregarea datelor (frontend-service). Aceste servicii vor comunica între ele prin apeluri HTTP.

Pe parcursul dezvoltării, se vor aplica mai multe șabloane de proiectare software (design patterns), precum Repository, Service Layer și Singleton, pentru a asigura separarea responsabilităților și o structură clară a codului. După implementare, vor fi realizate teste de performanță și de analiză a complexității codului, pentru a evalua impactul fiecărui model arhitectural. Rezultatele obținute vor fi analizate comparativ, urmărind identificarea avantajelor și limitărilor fiecărei abordări în contextul unei aplicații web reale, dar ușor de testat local.
